##  Important Linux Directories (Must Know)

### `/`
- Root of the filesystem
- Everything starts here

---

### `/bin`
- Essential user binaries (commands)
- Examples:
  - `ls`
  - `cp`
  - `mv`
  - `cat`

> These commands are available even in minimal or recovery mode.

---

### `/etc`
- System configuration files
- Examples:
  - `/etc/passwd`
  - `/etc/shadow`
  - `/etc/hosts`
  - `/etc/ssh/sshd_config`

---

### `/home`
- Home directories for users
- Example:
  - `/home/user1`

---

### `/root`
- Home directory of the **root user**
- Only accessible by root

---

### `/var`
- Variable data (changes frequently)
- Examples:
  - Logs: `/var/log`
  - Web data: `/var/www`

---

### `/usr`
- User-installed programs and libraries
- Not user-specific
- Contains:
  - `/usr/bin`
  - `/usr/lib`

---

### `/tmp`
- Temporary files
- World-writable by default

---

##  Manual Pages (`man` Command)

The `man` command displays **official documentation** for Linux commands, files, and system calls.

Example:

```bash
man ls
```

---

##  Manual Page Sections (Very Important)

Man pages are divided into **numbered sections**.

### Common Sections You Must Know

| Section | Meaning                        |
| ------- | ------------------------------ |

| 2       | System calls                   |
| 3       | Library functions              |
| 4       | Device files                   |
| 5       | Configuration files            |
| 6       | Games                          |
| 7       | Miscellaneous                  |
| 8       | System administration commands |

##  Specifying a Man Page Section

##  Linux Command Structure

General command syntax:

```bash
command [options] [arguments]
````

### Example:

```bash
ls -la /etc
```

| Part   | Meaning                              |
| ------ | ------------------------------------ |
| `ls`   | Command                              |
| `-la`  | Options (flags that modify behavior) |
| `/etc` | Argument (target of the command)     |

---

##  Options vs Arguments

### âœ… Options

* Modify **how** a command behaves
* Usually start with `-` or `--`

Examples:

```bash
ls -l
ls --all
```

---

### âœ… Arguments

* Specify **what** the command acts on
* Files, directories, users, IPs, etc.

Examples:

```bash
ls /var/log
cd /home/user
```

##  Executing Multiple Commands in One Line

### Using `;` (Always execute)

```bash
cmd1 ; cmd2
```

---

### Using `&&` (Only if first succeeds)

```bash
cmd1 && cmd2
```

---

### Using `||` (If first fails)

```bash
cmd1 || cmd2
```

`Ctrl + D` means **End Of File (EOF)**

* Logout

