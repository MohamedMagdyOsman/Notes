## 3ï¸âƒ£ Links in Linux

### ğŸ”¹ Symbolic (Soft) Link

A **symbolic link** is a file that points to another file or directory.
It is similar to a **Windows shortcut**.

```bash
ln -s /path/original /path/link
```

* Has a **different inode**
* If the original file is deleted, the link **breaks**

---

### ğŸ”¹ Hard Link

A **hard link**:

* Shares the **same inode** as the original file
* Points directly to the fileâ€™s data
* Has a different filename but identical content

```bash
ln /path/original /path/link
```

If the original file is deleted:

* The data still exists as long as **one hard link remains**

## 4ï¸âƒ£ Searching for Files

### ğŸ”¹ `locate` Command

`locate` is a **very fast** file search command.

* Uses a database (not live search)
* Database is updated by **root**
* Newly created files may not appear immediately
* Searches **only by filename**

```bash
locate passwd
```

---

### ğŸ”¹ `find` Command

The `find` command searches **live in the filesystem**.

It can search based on:

* Name
* Size
* Time
* Owner
* Permissions
* Type

```bash
find /path [options]
```

#### Common `find` Options

```bash
-name filename
-size [+|-]n
-atime [+|-]n
-mtime [+|-]n
-user loginID
-type
-perm
```
